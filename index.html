Product Concept Summary

What the system is, what problem it solves, and why current systems fail

Core Problem: Current DON personnel systems are optimized for administrative transactions — filling billets, processing orders, tracking pay. They produce no native intelligence about capability. A commander cannot query "which units have the most critical skill gaps relative to their assigned mission" without weeks of manual analysis across five disconnected systems.

The Intelligence Gap

TFMMS knows what billets are authorized. NSIPS knows who is assigned. Waypoints knows what training was completed. No system connects these into a capability picture. The gap between what a unit needs and what it actually has exists in spreadsheets, tribal knowledge, and post-deployment lessons learned — not in an authoritative system.

The Strategic Cost

When the POM changes, when a mobilization order drops, or when a disruptive technology enters the force, manpower planners are operating blind. Decisions about MOS/NEC realignment, training investment, contractor substitution, and cross-component utilization are made on stale, incomplete, manually-compiled data — at the speed of bureaucracy, not operations.

TIA Core Thesis: The intelligence layer must be built above existing transaction systems — not replacing them, but fusing their outputs into decision objects: match scores, readiness risks, substitution options, and demand forecasts. The architecture federated, not consolidating. The data stays where it lives; the intelligence is what moves.

Three Types of Gaps TIA Solves

  • Authorization Gap: Requirement exists, billet not authorized. Policy/budget problem.
  • Execution Gap: Billet authorized but unfilled. Recruiting/assignment problem.
  • Capability Gap: Billet filled, but person lacks required skills. Training/assignment problem.

Each gap type maps to a different policy lever. Current systems cannot distinguish between them at enterprise scale.

Design Principles

  • Federated — own the intelligence layer, not the data
  • Non-disruptive to source systems
  • USMC-first, DoN-scalable
  • Role-based access with analyst and executive views
  • Compatible with AI/ML augmentation from day one
  • Decision objects, not dashboards for their own sake
  • Cross-service crosswalk (NEC ↔ MOS ↔ OFC/AFSC) built in

Who Uses It

  • Commanders / G1/N1: Unit capability picture, gap warnings
  • Manpower Planners (M&RA): Enterprise gap analysis, POM support
  • Training Commands: Demand signals for pipeline shaping
  • Force Developers: Scenario and modernization planning
  • SYSCOM/DC I&L: Industrial base and contractor workforce analysis
  • CNO/CMC Staff: Strategic readiness picture

Entity Relationship Model / Ontology

The semantic layer that transforms transaction data into capability intelligence

Design Intent: Current systems model talent as position-filling (person → billet). TIA models talent as capability-matching (person's skill profile → mission's capability requirement). This requires a richer ontology layer, built above existing schemas, that the source systems never need to understand.

Critical Design Note — Gap as Derived Entity: Gap is computed, not stored as static data. However, gap snapshots ARE persisted as time-series records to enable trend analysis (Historical Variance Trends, per the challenge questions). The computation runs on the live ontology; the snapshot feeds the trend store. This prevents stale gap data from being treated as authoritative while still enabling historical analysis.
Critical Design Note — Scenario Engine: Scenarios apply delta-modifiers on top of the base ontology — they never overwrite base data. A POM cut scenario creates a shadow requirement set and shadow risk scores without touching the live data. Multiple scenarios can run simultaneously with independent result sets, enabling direct comparison.

MVP Feature List

Six capability modules mapped to the six required outputs and challenge questions

F-01 · Capability Inventory

OUTPUT-1

Real-time inventory of qualified personnel by skill, MOS/NEC, unit, service, and component. Supports queries such as "How many Marines in I MEF are qualified in [skill X] at proficiency level 3+?" Cross-component view shows Navy and Marine Corps inventory side-by-side against shared mission requirements.

TFSMS · MCTFS · TFMMS · NSIPS Phase 1 MVP Analytics: Count, Distribution, Filter

F-02 · Skill Demand Map

OUTPUT-2

Maps skill demand to mission priorities and strategic direction (SECWAR, SECNAV, CNO, CMC guidance). Shows which skills have rising demand vs. current supply trends. Flags emerging capability needs for disruptive technologies. Outputs a heatmap of demand intensity by skill family over 1, 3, and 5-year horizons.

EIPL · MFTs · TFMMS · TFSMS · Strategy Signals Phase 2 Analytics: Time-series, Forecast, Trend

F-03 · Gap Analysis Engine

OUTPUT-3

Three-dimensional gap analysis: Authorization gaps (requirement vs. billet), Execution gaps (authorized billet vs. filled billet), Capability gaps (assigned person vs. required skill). Distinguishes between these gap types at unit, command, and enterprise level. Shows Work Executed vs. Planned discrepancies. Identifies execution drivers — why funded billets remain unfilled.

TFMMS · TFSMS · NSIPS · MCTFS · PBIS Phase 1 MVP — Core Feature Analytics: Gap computation, Root-cause classification

F-04 · Risk Scoring

OUTPUT-4

Quantified operational risk score for each identified gap. Risk = f(gap magnitude × mission criticality × time-to-close × substitutability). Surfaces which capability gaps have highest operational risk if left unaddressed. Enables prioritized resourcing decisions. Integrates readiness outcomes as feedback to validate risk model accuracy over time.

All gap data + Mission criticality weights Phase 1 MVP Analytics: Composite scoring, Ranking, Sensitivity analysis

F-05 · Interoperability Recommender

OUTPUT-5

Cross-service and cross-component substitution analysis. Given a capability gap in a Navy unit, identifies USMC personnel with equivalent or substitutable skills via the NEC↔MOS crosswalk, with a match score and constraint flags (security clearance, deployment eligibility, training currency). Enables cross-assignment, augmentation, or training pipeline recommendations before a gap becomes a crisis.

CrosswalkMapping · NSIPS · MCTFS · DCPDS Phase 2 Analytics: Match scoring, Constraint filtering, Recommendation ranking

F-06 · Scenario Analysis Module

OUTPUT-6

Non-destructive scenario overlays for POM changes, mobilization orders, or modernization shifts. Planners define a scenario (e.g., "10% POM reduction in aviation MOS billets"), and the system computes resulting gap deltas, risk score changes, and affected units — against the current baseline. Multiple scenarios can run in parallel. Budget Vulnerability analysis shows which billets are most exposed when the POM changes and why.

Scenario Engine · Gap Engine · All source data Phase 2/3 Analytics: What-if simulation, Delta comparison, Sensitivity analysis

Sample User Stories

Grounded in the challenge questions — each story maps to a specific decision need

USMC G1 Manpower PlannerUS-01 · Gap Analysis · F-03

"As a G1 manpower planner, I need to identify which MOS fields have the largest gap between authorized billets and qualified personnel assigned, so that I can prioritize accession and retention actions for the next POM cycle."

Acceptance: System displays authorization, execution, and capability gaps by MOS at the MEF level, with drill-down to battalion. Gap type is clearly distinguished. Export to PBIS-compatible format is available.

Navy N1 Workforce PlannerUS-02 · Risk Scoring · F-04

"As an N1 planner, I need to know which NEC shortfalls pose the highest operational risk to fleet readiness right now, ranked by severity, so that I can brief COMNAVPERS on where to direct emergency accession authorities."

Acceptance: Risk-ranked gap list with composite score, mission impact linkage, and time-to-close estimate. Filters by fleet, command, and readiness tier. Exportable to brief format.

Force Development AnalystUS-03 · Demand Forecasting · F-02

"As a force developer, I need to understand which skill sets will face the largest demand increases over the next 3–5 years based on CMC and CNO strategic priorities, so that training pipelines can be shaped now rather than after the gap materializes."

Acceptance: Skill demand forecast with source traceability to strategy documents. Shows delta between current pipeline output and projected demand. Flags emerging capability requirements for disruptive technology fields.

ASN M&RA POM AnalystUS-04 · Scenario Analysis · F-06

"As a POM analyst, I need to model the impact of a proposed 8% reduction in naval aviation billet lines against current readiness data, so that I can provide SECNAV staff with a quantified risk assessment before the decision is finalized."

Acceptance: Scenario tool allows billet line reduction inputs. System computes gap deltas, risk score changes, and affected unit list within 5 minutes. Scenario can be saved, named, and compared to baseline and alternative scenarios.

SYSCOM Industrial Base AnalystUS-05 · Interoperability · F-05

"As a SYSCOM workforce analyst, I need to know where workforce shortages in the industrial base are impacting ship maintenance timelines, and whether military personnel with equivalent skills could augment civilian and contractor shortfalls."

Acceptance: Industrial base gap view linked to FPDS-NG contractor data. Interoperability recommender identifies military personnel with crosswalk-matched skills, with constraint flags for eligibility. Recommendation includes match confidence score.

MEF Commanding GeneralUS-06 · Capability Inventory · F-01

"As a MEF commander, I need a single-page view of my command's current capability against its assigned missions — showing where I am green, amber, and red — so that I can make informed deployment and employment decisions without waiting for a J1 brief."

Acceptance: Commander view shows unit capability heatmap by mission area. Color-coded against assigned mission requirements. Drill-down available but not required for top-level situational awareness. Refreshes daily from source systems.

Training Command SchedulerUS-07 · Skill Demand · F-02

"As a training command scheduler, I need to see which courses have the largest mismatch between pipeline capacity and projected demand, so that I can make a business case for additional course seats or instructor billets in the next fiscal year."

Acceptance: Training demand vs. pipeline capacity view by course/MOS/NEC. Shows projected shortfall with confidence interval. Exportable with source data citations for POM submission support.

Three-Phased Roadmap

Prototype → Pilot → Enterprise Scale — USMC-first, DoN-scalable

Sequencing Logic: Start with USMC because TFSMS and MCTFS are a tighter, more coherent data environment than the Navy's distributed systems. Prove the intelligence layer works at one-service scale. Then use the Navy Total Force Data Hub (already harmonized on Jupiter/Advana) to accelerate Navy integration rather than rebuilding harmonization from scratch.
Phase 1

Prototype

Months 0–6 · USMC Focus · ~$2–4M
  • Ingest TFSMS + MCTFS + Waypoints
  • Build core ontology and TIA knowledge graph
  • Implement F-01 Capability Inventory
  • Implement F-03 Gap Analysis (3 gap types)
  • Implement F-04 Risk Scoring (v1)
  • Build initial NEC↔MOS crosswalk (manual, 200 high-priority skill pairs)
  • Deploy to single MEF as test user community
  • Analyst workbench + Commander view (static export)
  • RBAC and PII tokenization framework
  • Validate gap model against known readiness outcomes
Phase 1 Exit Gate TIA identifies ≥3 critical capability gaps not surfaced by current manual processes, validated by MEF G1. Gap type classification accuracy ≥85%.
Phase 2

Pilot

Months 6–18 · Navy + USMC · ~$8–15M
  • Extend to TFMMS + NSIPS (Navy component)
  • Integrate Navy Total Force Data Hub (Jupiter/Advana)
  • Add FPDS-NG for industrial base analysis
  • Add DCPDS for civilian workforce
  • Implement F-02 Skill Demand Map
  • Implement F-05 Interoperability Recommender
  • Implement F-06 Scenario Analysis (POM + mobilization)
  • Deploy to ASN M&RA staff + 2 Fleet/Force commands
  • Executive dashboard (live, not static export)
  • Expand crosswalk to 2,000+ skill pairs with ML assistance
  • Integrate EIPL/MFT mission-capability linkage
  • First POM cycle integration — brief SECNAV staff with TIA-generated analysis
Phase 2 Exit Gate TIA-generated scenario analysis is used in at least one formal SECNAV/CMC decision brief. Interoperability recommender demonstrates ≥1 validated cross-service substitution recommendation.
Phase 3

Enterprise Scale

Months 18–36 · Full DON · ~$20–40M
  • Full DoN integration — all components and reserves
  • Add ERP + DJMS for full financial integration
  • Add Census data for recruiting pipeline modeling
  • ML-powered demand forecasting (F-02 v2)
  • Automated risk alerting and anomaly detection
  • Real-time crosswalk maintenance with ML confidence scoring
  • Predictive assignment optimization (constraint-aware)
  • DoD IL4+ authorization and ATO
  • API layer for external system consumption
  • Feedback loop: Readiness Outcomes → Risk model recalibration
  • Expand toward Joint (OSD/JS staff) capability on request
  • Training pipeline integration for demand-driven course scheduling
Phase 3 Exit Gate TIA is the authoritative source for enterprise capability gap analysis used in POM build. Time-to-produce gap analysis reduced from weeks to hours. Adopted by ≥80% of target user community.
Critical Risk — Crosswalk Maintenance: The NEC↔MOS crosswalk will degrade over time as new NECs and MOSs are created, retired, or redefined. Phase 2 must establish a governed maintenance process — not just a one-time build. This is the single most likely failure mode for the interoperability use case.
